# Information Design · 信息设计

> Edward Tufte 等先驱的信息设计原则，应用于展示层。

## 1. 数据墨水比 (Data-Ink Ratio)

**原则**：最大化数据墨水/总墨水比 — 删除不传递信息的装饰。

**应用**：
- 删除不必要的网格线、边框、背景色
- 只用颜色传递信息，不是装饰
- 表格数据用最小边框

```css
/* ✅ 高数据墨水比表格 */
.data-table th,
.data-table td {
  border: none;
  border-bottom: 1px solid var(--hr);
  padding: 8px 16px;
}

/* ❌ 低数据墨水比：过多边框和背景 */
.data-table th,
.data-table td {
  border: 1px solid #ddd;
  background: #f9f9f9;
  padding: 8px;
}
```

## 2. 图表完整性

- 坐标轴从 0 开始（防止视觉歪曲）
- 长宽比反映数据比例
- 不使用 3D 图表（歪曲比较）

## 3. 信噪比 (Signal-to-Noise)

**原则**：最大化信号（信息）与噪声（装饰）的比例。

**应用**：
- Stats 布局不用装饰背景，数字本身是英雄
- 正文区域不用渐变/纹理
- Minimal 风格的哲学基础

## 4. F 型阅读模式

**原则**：用户浏览内容呈 F 型模式（先横读顶部，再向下扫左侧）。

**应用**：
- 最重要的信息在左上
- 标题需要抓住注意力
- 列表/要点用醒目项目符号

## 5. 倒金字塔结构

**原则**：最重要的信息在最前面，然后是支持细节，最后是背景。

**应用**：
- 标题给出结论
- 副标题给出关键数据
- 正文展开细节

## 6. 格式塔原则图版布局

→ 见 `gestalt.md` — 所有 6 个原则都直接应用于排版布局。

## 7. 排版信息层级

文字大小应区分出信息层级，让读者一眼判断优先级：

```
H1 (封面) → 品牌/主张 （最重要的信息）
H2 (章节) → 新概念/新章节
H3 (区块) → 子主题
正文       → 详细内容
元数据     → 日期/作者/来源（最不重要）
```
