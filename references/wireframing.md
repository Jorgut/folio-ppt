# Wireframing for Presentations

> 从 lo-fi 到 hi-fi 的演示文稿线框图方法

## 三级 Fidelity 定义

### Lo-fi Wireframe（低保真线框）

**目的**：快速探索结构，测试布局概念

**特征**：
- 方块 + 占位符，灰度
- 不关心颜色、字体、间距
- 5-10 分钟做一页
- 可以手画

**在 Folio 中的实现**：

```
┌─────────────────────────────┐
│                             │
│  ┌─────────┐  ┌──────────┐ │
│  │  HERO   │  │  IMAGE   │ │
│  │  TEXT   │  │  PLACE   │ │
│  │         │  │  HOLDER  │ │
│  └─────────┘  └──────────┘ │
│                             │
│  ┌──────────────────────┐   │
│  │    BODY TEXT AREA    │   │
│  └──────────────────────┘   │
│                             │
│         META / FOOTER       │
└─────────────────────────────┘
```

**Lo-fi 检查清单**：
- [ ] 每页只有 1 个核心信息？
- [ ] 信息层级清晰（大 → 中 → 小）？
- [ ] 图片位置合理（不挡文字）？
- [ ] 元数据在安全区内（不碰导航）？

### Mid-fi Wireframe（中保真线框）

**目的**：验证结构，获得 stakeholder 反馈

**特征**：
- 定义好的 grid + 标注
- 内容层级清晰
- 响应式断点考虑
- 1-2 天完成

**在 Folio 中的实现**：

```
┌──────────────────────────────────────────┐
│ safe-x ←── content area ──→ safe-x       │
│ ┌──────────────────────────────────────┐ │
│ │  grid: split 4-8                     │ │
│ │  ┌──────┐  ┌────────────────────┐   │ │
│ │  │      │  │                    │   │ │
│ │  │ Text │  │     Image          │   │ │
│ │  │ Col  │  │     16:10          │   │ │
│ │  │ 4/12 │  │     100% width     │   │ │
│ │  │      │  │                    │   │ │
│ │  │      │  └────────────────────┘   │ │
│ │  │      │                           │ │
│ │  │      │  Caption: mono 12px       │ │
│ │  └──────┘                           │ │
│ │                                      │ │
│ │  Nav safe: 72px padding-bottom      │ │
│ └──────────────────────────────────────┘ │
│ ┌──────────────────────────────────────┐ │
│ │  Nav: prev → dots → count → share   │ │
│ └──────────────────────────────────────┘ │
└──────────────────────────────────────────┘
```

**Mid-fi 标注要素**：

| 标注项 | 内容 |
|--------|------|
| Grid 布局 | `split 4-8` / `editorial 2-col` / `gallery auto-fill` |
| 文字层级 | `hero: 6.4vw serif` / `lead: 1.35rem sans` / `body: 1.125rem sans` |
| 图片比例 | `16:10` / `4:3` / `3:2` / `21:9` |
| 间距 | `--sp-6: clamp(16px, 1.8vw, 32px)` |
| 安全区 | `safe-x: clamp(16px, 5vw, 80px)` |
| 导航安全 | `padding-bottom: 72px` |

### Hi-fi Wireframe（高保真线框）

**目的**：开发交接，最终审批

**特征**：
- 真实内容（不是 lorem ipsum）
- 精确间距（基于 clamp() 系统）
- 响应式断点完整
- 交互标注（动画、hover 状态）
- 1-2 周完成

**Hi-fi = 最终 Mockup 的骨架**，只差颜色和图片。

## Wireframe → HTML → Native PPTX 流程

```
Step 1: Lo-fi wireframe（手画 or 简单数字）
        ↓ 确认结构
Step 2: Mid-fi wireframe（定义 grid + 标注）
        ↓ 确认布局
Step 3: Hi-fi wireframe（真实内容 + 精确间距）
        ↓ 确认内容
Step 4: HTML 实现（用 Folio 模板 + CSS 变量）
        ↓ 测试渲染
Step 5: Native PPTX 导出（所有文字 native，图片只用于照片）
        ↓ 测试在 PowerPoint/Google Slides/Keynote
Step 6: 交付
```

## Wireframe 验证检查清单

每级 fidelity 完成后都要检查：

### Lo-fi 检查

- [ ] 每页只有 1 个核心信息？
- [ ] 信息层级清晰？
- [ ] 叙事弧完整（Hook → Core → Takeaway）？
- [ ] 没有多余的页？

### Mid-fi 检查

- [ ] Grid 布局已定义？
- [ ] 文字层级已标注？
- [ ] 图片比例已指定？
- [ ] 安全区已考虑？
- [ ] 导航安全区已预留？
- [ ] 响应式断点已考虑？

### Hi-fi 检查

- [ ] 真实内容（不是 lorem ipsum）？
- [ ] 字号基于 clamp() 系统？
- [ ] 间距基于 spacing tokens？
- [ ] 字体分工正确（serif/sans/mono）？
- [ ] 主题节奏合理（hero/light/dark 交替）？
- [ ] 动画标注已添加？
